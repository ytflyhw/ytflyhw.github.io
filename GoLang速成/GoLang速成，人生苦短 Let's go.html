GoLang速成，人生苦短 Let’s go

1、基本原则

  • GoLang 是一门编译型语言，程序员写的代码将会被直接编译为可执行文件（没有Python一类的解释器）

  • 开发需要配置GOPATH，这是之后开发的根目录

  • 所有程序一定属于某个package，需要在程序的最初进行声明。main包下的main函数将是整个程序的入口。

  • 所有 import 了的包，函数内声明的变量都必须被使用，否则编译将会报错。

  • 不必在意缩进，空格等对齐问题，内置的 go fmt 将会为我们处理好一切

  • 函数外只能写标识符的声明，不能写任何语句。标识符只能以字母或下划线开始。

  • GoLang 所有的参数传递默认都是值传递，如果希望处理变量本身，需要指明参数类型为指针，但函数内不需要可以强调指针。

  • GoLang通过起多个goroutine实现多线程，不同线程之间不同于 JAVA 的共享内存对数据加锁的方式实现同步，而是通过 Channel（类似消息队列，一个线程往里写，一个线程往外读） 实现共享内存。
